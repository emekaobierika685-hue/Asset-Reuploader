Changes Summary for Making Sound Reuploading Free

Backend Changes (Go):

1. Created new file: internal/roblox/ide/upload_sound.go
   - Added NewUploadSoundHandler function similar to NewUploadAnimationHandler
   - Uses IDE publish endpoint: https://www.roblox.com/ide/publish/UploadNewSound
   - Handles authentication, token validation, and inappropriate name errors
   - Returns asset ID directly as int64

2. Updated internal/app/assets/sound/sound.go
   - Changed import from "roblox/publish" to "roblox/ide"
   - Modified uploadHandler call from publish.NewUploadAudioHandler to ide.NewUploadSoundHandler
   - Updated error handling to use ide.UploadSoundErrors instead of publish.UploadAudioErrors
   - Removed quota exceeded error handling since free uploads don't have limits
   - Fixed return value handling since IDE handler returns int64 directly

Plugin Changes (Luau):

- No changes required in plugin/src/App/Tabs/Sound.luau or plugin/src/App/ReuploadIds.luau
- The plugin already sends "Sound" as assetType, and the backend now handles it via the free IDE upload

Result:
- Sound reuploading now uses the same free IDE publish endpoint as animation and mesh
- No Robux charges or payment required
- Consistent error handling and retry logic
- Compilation successful with no errors
