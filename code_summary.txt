Code Summary for Asset-Reuploader Workspace

Go Files (Backend/CLI):

- cmd/assetreuploader/assetreuploader.go: Serves as the main entry point for the CLI application. It handles Roblox cookie authentication and starts a local HTTP server for asset reuploading operations. Role: Initializes the application and manages authentication flow.

- cmd/assetreuploader/router.go: Sets up HTTP routes and handlers for the local server. It processes reupload requests, manages response history, and handles JSON export of results. Role: Provides the web API interface for the plugin to communicate with the backend.

- internal/app/assets/assets.go: Acts as the central dispatcher for asset reuploading operations. It routes requests to specific asset type handlers (animation, mesh, sound) and manages the overall reupload workflow. Role: Orchestrates asset-specific reuploading logic.

- internal/app/assets/animation/animation.go: Implements the logic for reuploading animation assets. It fetches existing animations, processes them, and uploads new versions to Roblox. Role: Handles animation asset reuploading with retry mechanisms and error handling.

- internal/app/assets/sound/sound.go: Manages the reuploading of sound assets. It retrieves sound files, applies necessary transformations, and uploads them to Roblox's servers. Role: Processes sound asset reuploads with permission checks and batch operations.

- internal/app/assets/mesh/mesh.go: Handles mesh asset reuploading. It downloads mesh data, processes it, and uploads modified versions. Role: Manages mesh asset reuploads with support for different mesh types.

- internal/app/assets/shared/assetutils/assets_info.go: Provides utilities for fetching asset information in batches from Roblox APIs. It handles chunked requests and retry logic for reliability. Role: Batch processing utility for asset metadata retrieval.

- internal/app/assets/shared/assetutils/batch_body.go: Defines structures and functions for creating batch request bodies for asset operations. Role: Data structure helper for bulk asset API calls.

- internal/app/assets/shared/assetutils/filter.go: Implements filtering logic for asset selection based on various criteria. Role: Asset filtering utility for selective reuploading.

- internal/app/assets/shared/assetutils/permission_body.go: Creates request bodies for asset permission operations. Role: Permission management helper for assets.

- internal/app/assets/shared/clientutils/get_new_cookie.go: Handles Roblox cookie authentication and renewal. It prompts users for new cookies when authentication fails. Role: Authentication utility for maintaining valid Roblox sessions.

- internal/app/assets/shared/clientutils/get_request.go: Provides HTTP request utilities with cookie handling. Role: HTTP client wrapper for authenticated requests.

- internal/app/assets/shared/permissions/permissions.go: Manages asset permissions and universe editing rights. Role: Permission checking and management for Roblox assets.

- internal/app/assets/shared/uploaderror/upload_error.go: Defines error types and handling for upload operations. Role: Error categorization for upload failures.

- internal/app/config/config.go: Manages application configuration settings. Role: Configuration loader and accessor.

- internal/app/context/context.go: Provides application context with client and response references. Role: Context management for request handling.

- internal/app/context/logger.go: Implements logging functionality for the application. Role: Logging utility for debugging and monitoring.

- internal/app/context/pause_controller.go: Controls application pausing and resuming for user interaction. Role: Pause/resume mechanism for interactive flows.

- internal/app/request/request.go: Handles request parsing and validation from raw inputs. Role: Request processing and validation.

- internal/app/response/response.go: Manages response collection and notification callbacks. Role: Response handling and history tracking.

- internal/console/input.go: Provides console input utilities, including clipboard access. Role: User input handling for CLI.

- internal/console/output.go: Handles console output and screen clearing. Role: Console display management.

- internal/atomicarray/atomic_array.go: Implements a thread-safe array for concurrent operations. Role: Concurrent data structure utility.

- internal/color/color.go: Provides colored output for console messages. Role: Color formatting for CLI output.

- internal/files/files.go: Offers file read/write operations. Role: File I/O utility.

- internal/retry/errors.go: Defines retry-related error types. Role: Error handling for retry logic.

- internal/retry/options.go: Configures retry behavior options. Role: Retry configuration utility.

- internal/retry/retry.go: Implements retry logic with backoff strategies. Role: Retry mechanism for API calls.

- internal/roblox/assetdelivery/batch.go: Handles batch asset delivery requests to Roblox. Role: Batch asset fetching client.

- internal/roblox/assets/permissions.go: Manages asset permissions via Roblox APIs. Role: Permission API client.

- internal/roblox/authenticate.go: Performs Roblox user authentication. Role: Authentication API client.

- internal/roblox/client.go: Core Roblox API client with cookie management. Role: Main API client for Roblox interactions.

- internal/roblox/games/groupGames.go: Retrieves game information for groups. Role: Group games API client.
